A GitHub account, along with W3C's testsuite-so-far

W3C uses GitHub to receive submissions and to host the complete testsuite, so please join GitHub if you haven't already.

Then, get the testsuite:

  • in your local terminal, cd into the directory in which you want to put the web-platform-tests structure
  • type git clone git@github.com:w3c/web-platform-tests.git

Go ahead and fork it, so that you'll be ready to submit tests:

  • up in GitHub, on the source page, choose fork, choose SSH, and copy the URL.
  • back in your local terminal, type git remote add <your_forkname> <URL>

Some JavaScript Files

Thanks to James Graham, there is a great API to help you write robust tests and automatically assemble the pass/fail results. Please download these files from their own GitHub repositories:

  • testharness.js
  • testharness-report.js
  • idlharness.js
  • webidl2.js

Save these files into a resources folder within whichever local directory you will use for writing your own tests
